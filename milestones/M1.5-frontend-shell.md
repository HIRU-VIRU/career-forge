> **Dependencies:** M1 complete (Bedrock, DynamoDB, S3, GitHub OAuth all live)

## Goal

Transform the existing frontend (currently branded "LaTeX Resume Agent", pointed at the old GCP backend) into the full CareerForge shell. Every feature page exists with real navigation, real auth, and production-ready empty states. No mocks — real API calls, real users. M2–M5 then only need to drop data into already-built UI.

### What Already Exists (don't rebuild)
- `src/app/page.tsx` — landing page (needs rebrand + copy overhaul)
- `src/app/login/page.tsx` — GitHub OAuth + email/password login (needs backend URL update)
- `src/app/dashboard/page.tsx` — tabbed dashboard shell (needs tab restructure + new tabs)
- `src/components/dashboard/` — jobs-list, resumes-list, projects-list, templates-list, profile-view
- Full Radix UI + Tailwind + React Query + Lucide stack already installed

### What Needs to Be Built
- CareerForge rebrand across all pages
- API client pointing to new FastAPI + DynamoDB backend
- Dashboard restructured: 5 tabs (Resume, Skill Gap, Job Scout, Apply, Profile)
- Skill Gap page shell with empty state
- Job Scout page shell with empty state
- Tailored Apply + Application Tracker page shells
- Proper loading skeletons and error states on all pages
- Dark mode + design system polish per Web Interface Guidelines
